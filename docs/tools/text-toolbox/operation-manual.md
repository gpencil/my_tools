# 文本工具箱操作手册

## 1. 本地启动

在仓库根目录执行：

```bash
npm install
npm run dev
```

默认会启动 Vite 本地服务，通常访问地址是：

```text
http://localhost:5173
```

如果 5173 被占用，Vite 会自动切换到其他端口，以终端输出为准。

## 2. 工具入口

打开页面后，当前默认会进入唯一工具模块：`文本工具箱`。

`文本工具箱` 顶部有两个标签：

- `JSON 格式化`
- `文本比对`

## 3. JSON 格式化

### 使用步骤

1. 把原始 JSON 粘贴到左侧 `原始 JSON` 编辑区。
2. 如果内容是合法 JSON，右侧 `结构化结果` 会立即显示格式化后的结果。
3. 如果内容不合法，页面会直接显示错误信息、行号、列号和 offset。

### 适用输入

- 单行 JSON
- 多行 JSON
- 对象、数组、嵌套结构

### 结果说明

- 右侧有行号，便于核对结构。
- 右侧是只读视图，不会回写到左侧。
- 合法 JSON 会被统一格式化为 2 空格缩进。

### 错误提示说明

如果提示解析失败，会显示：

- `errorMessage`：错误类型
- `line`：出错行
- `column`：出错列
- `offset`：在原始文本中的字符偏移位置

## 4. 文本比对

### 使用步骤

1. 把第一份文本粘贴到左侧 `左侧文本`。
2. 把第二份文本粘贴到右侧 `右侧文本`。
3. 页面会实时按行重新计算差异。
4. 查看颜色标记和同步滚动后的对齐结果。

### 当前比对规则

- 以“行”为单位比较。
- 比较前会先去掉每行首尾空格。
- 如果一侧中间多出一行，只标记那一行，不让后续整片错位误报。
- 如果左右某一行都存在，但内容不同，会标记为 `modified`。

### 颜色说明

- 左侧独有：左侧暖色背景
- 右侧独有：右侧绿色背景
- 修改行：两侧琥珀色背景
- 对齐空白区：用于让后续真实相同行重新对齐，不代表额外一条文本内容

### 浏览说明

- 左右滚动默认同步。
- 两边都带行号，方便逐行核对。

## 5. 已知边界

当前第一版明确不做以下内容：

- 文件导入导出
- 剪贴板历史
- 非法 JSON 自动修复
- 行内逐字 diff
- 本地持久化
- 多工具之外的桌面打包

## 6. 仓库约定

`文本工具箱` 是 `my_tools` 主应用里的第一个工具模块。

后续新增工具时，按同样方式补三类内容：

1. 工具代码
2. README 中的工具索引
3. `docs/tools/<tool-name>/` 下的操作手册
